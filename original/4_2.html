The MINT System described in this proposal is technically a Intranet solution, that fits perfectly in ACME's infrastructure. The MINT-Application is a scaleable open environment and can be easily extended especially over the Internet.

The client-server architecture of the proposed solution is divided into four different layers. The layers are the database server layer, the Application Server layer, the Replication Server layer that is transparent to the user, and the Client or Browser layer.

MINT works in a environment that is divided into three main areas.

  • The ACME world, which can be accessed to obtain data (e.g. marketing packages, vehicle location, financial quotes) or to trigger functions (e.g. validate configuration or order new vehicle).
  • The dealer server world, which stores all dealer specific data (e.g. customer data, used vehicles) and provides some functionality similar to the ACME world (e.g. locate new vehicle at dealer, or in dealer group stock).
  • The client world is divided into a dealer-side area and a public area. On the dealer side area are the client computers used by the sales consultants and the showroom application computers. The laptop computers of the sales consultants are also part of this area. The dealer-side area is subject to specific adaptations and allows the dealer to add his own corporate identity. The public area contains the public kiosk versions.

The parts marked DS (DealerSystem) and AS (ACME-System) are the corresponding ACME and dealer legacy systems that are accessible by using appropriate middleware.

The main access path for this system is marked in red. It originates from the client computer, which accesses the ACME or dealer data and functions in the course of the sales process. These functions and date, that resides on different systems, servers and locations are wrapped logicaly by MINT-Business Objects. These business objects are the key to encapsulate complex ACME business rules. The application server handles the logic of the process. The replication server remains transparent.

The browser approach as well as this architecture have two major advantages of great importance:

  • The browser offers maximal flexibility to incorporate new developments in the multimedia market during the project and afterward without the need for additional development. This is a guarantee that the application will remain state-of-the-art through its foreseen lifespan.
  • The browser offers the possibility of separating the creative design of layout and multimedia objets from the application that incorporates business and process rules.

The basic network protocol stack for communication used by all layers of MINT is TCP/IP, as requested by ACME. All layers will be implemented as 32bit applications or services.
