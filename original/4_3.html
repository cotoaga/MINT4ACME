4.3.1. Browser (Client)

The Browser (client layer) is the only visible top layer for all classes of users. The lower layers are always transparent to all users. This assures that the system is viewed by everyone and at every time as one logical device. The Browser visually wraps (look & feel) the different systems that are available to the customer or to the sales consultant. The display and layout is designed to be totally customer facing.

The browser behaves at the same time as an environment manager and as a business application. It offers the proposed key capabilities:

  • Single point of access to distributed systems
  • Application selection based on data
  • Intuitive GUI
  • Integrated help system
  • Transparent integration of Microsoft/Unix applications

The Browser uses HTTP (Hyper Text Transfer Protocol) and TCP/IP to access lower layers. The normal access (left hand side) goes through the replication server to the application server. This way is relevant for the Dealer-MINT, the Laptop-MINT version in online mode and for the Kiosk-MINT version. The second way is a direct public access to the dealer system from the Internet. The third way (right hand side) is a direct public access from the Internet or public kiosks to the ACME-Web. All components of the MINT multi-tier architecture are sharing TCP/IP and HTTP as the common protocols.

The requested features that are visible to users are

  • ease of use,
  • integrated sales process,
  • a state-of-the-art multimedia display,
  • different layout and branding for different countries,
  • partial layout configuration (dealer or dealer group).

It is possible to use the Internet Explorer from Microsoft as well as the Netscape Communicator that are both available for Windows95 and Windows NT. These easy-to-use applications already have most of the required features and capabilities:

  • The display is controlled by structural layout done with HTML (Hyper Text Markup Language). This widely used new document standard is derived from SGML (Structured General Markup Language). It is the standard for Internet documents and makes different branding and layout in different countries possible and easy.
  • It offers a flexible handling of multimedia content. This includes: built-in display abilities for all requested multimedia objects (MMO) types, alternative displays if the specified MMOs are not available, dynamic re-sizing and adaptation of the layout depending on the available display area and easy adaptation to new MMO types by adding the necessary new ActiveX controls or Plug-ins.
  • The browser can be configured for different use cases like the Dealer-MINT or the Kiosk-MINT version.
  • At the end of the configuration process the browser can print a quotation containing pictures and the complete breakdown of costs and financing options.
  • The browser is perfectly suited for reuse of existing multimedia objects, advertising content and design that has already been done by other creative companies. This material can be distributed centrally by ACME.
  • The Browser acts as a pure display utility (screen & printer) without any business-specific logic. It encapsulates no process logic and is free of complex business rules owned by ACME.
  • The Browser has a low impact on hardware or software requirements.
  • Due to the wide spread use of browsers, it offers all users the same GUI for the different versions of the MINT-Application independent of the current location.

4.3.2. Replication Server

The Replication Server is a transparent layer between the client and application layer hiding the complex problem of working off-line to the client layer.

It also reduces network traffic, increases access speed for large multimedia objects (MMO) and supports the distribution and maintenance of the different layout & MMOs versions throughout the ACME dealer area.

The Replication Server is an intelligent cache server that is fully integrated in the system. The main difference to ordinary cache servers is that it is not only meant to optimize the information flow in one direction. It can be triggered from the client and also from the server side. When it is triggered from the Application Server it flushes cached orders and access statistics. When it is triggered from ACME side it starts updating its own database by fetching new versions of the objects. The Replication Server helps reducing network load by updating only new versions.

The Replication Server is a Win95/WinNT service and uses HTTP and TCP/IP for the communication with other parts of the system. It is based on the Java BOOSTER technology. Some of the operations (user or operator triggered) of the replication server are:

  • Unconditional updating of the entire cache. It would be part of system management function executed by a telnet access to the dealer server.
  • Loading pre-configured vehicles in stock
  • Loading historic configurations of selected customers
  • Loading marketing offers
  • Retrieving statistical data on the most frequently used areas and pages

4.3.3. Application Server

The Application Server is the integrating shell that logically wraps the different systems. This layer steers the identified Business Objects through the sales process acquiring more and more data. The application server retrieves this information by directly accessing ACME or dealer data through the proposed interfaces. The Application server can be installed on both WindowsNT and Unix platforms.

The Application Server maps the relevant business processes directly into HTML-documents that can be displayed by the Browser. The Application Server is the owner of the vehicle configuration data assigned to customers and the statistical information which is also stored on the dealer server. It also accumulates the statistical data obtained by the Replication Server on the different client computers.

The Application Server caches orders that are given in off-line mode and forwards them to ACME when the network is available (when the Replication Server can be reached from the Application Server).

Some of the operations (user-triggered) of the application server are:

  • Uploading of off-line orders when the network connection from the dealer server to the laptop clients is available.
  • Uploading of accumulated statistical data that can be retrieved from ACME.
  • Accessing dealer data from ACME.

4.3.4. ACME-Web (special Application Server)

This component is the requested public kiosk. It is implemented as a special version of the Application Server and offers dealer showroom MINT functions. Ideally, this is a country specific ACME-Web Server acting as a central online dealer. The main difference to the Dealer versions may be the amount and type of data that will be used. Normal customers still do not have enough bandwidth to get data intensive MMOs like MPEGs that also require special devices to play back fast enough, but this limitation is likely to disappear in the future.

The ACME-Web version can also be installed as a public kiosk version. It can be controlled remote from ACME. Just as the central ACME-Web its data can be used and forwarded to selected dealers to take contact to leads or prospects.

4.3.5. Database Layer

This layer represents all interfaces to ACME and dealer data that are available on the dealer server. The data is accessed from the Application Server by using ACME-Systems (AS) or Dealer-Systems (DS) functions.

The database layers encapsulate data and functions of AS and DS. MINT must rely on functions provided by those systems.
